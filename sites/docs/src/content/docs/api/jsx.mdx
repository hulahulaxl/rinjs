# jsx / jsxs

The `jsx` and `jsxs` functions are the heart of Rin's JSX runtime. They transform JSX syntax into lightweight Virtual Node objects.

## Usage

You rarely call these functions directly; they are mostly generated by the TypeScript compiler.

```tsx
import { jsx } from "rin-lib";

const node = jsx("div", {
  class: "container",
  children: "Hello World"
});
```

### Parameters

- **type**: The element name (string) or a Rin Component (function).
- **props**: An object containing attributes and the `children` array.
- **key**: (Optional) A unique identifier for the element, used for efficient DOM patching.

## Differences from React

Unlike React, Rin's `jsx` runtime:
1.  Does not use a global React object.
2.  Supports the standard `class` attribute natively.
3.  Does not perform automatic camelCase to kebab-case conversion.
